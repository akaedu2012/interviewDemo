# 无限循环问题修复说明

## 🐛 问题描述

**症状**:
- 浏览器控制台频繁调用 `/api/candidates` API（每隔几毫秒一次）
- 页面不断闪动和重新渲染
- 开发服务器日志显示大量 200 响应

**根本原因**:
React `useEffect` 无限循环 - `selectedSkills` 数组引用问题

## 🔍 问题分析

### 原始代码问题

```typescript
// ❌ 问题代码
const selectedSkills = searchParams.get("skills")
  ? searchParams.get("skills")!.split(",").filter(Boolean)
  : [];

const fetchCandidates = React.useCallback(async () => {
  // ... fetch logic
}, [page, pageSize, sortBy, sortOrder, searchKeyword, selectedSkills]);
//                                                        ^^^^^^^^^^^^^^
//                                                        每次都是新数组！

React.useEffect(() => {
  fetchCandidates();
}, [fetchCandidates]); // fetchCandidates 不断变化
```

### 问题流程

1. 组件渲染 → 从 URL 解析 `selectedSkills` 数组
2. 每次渲染都创建**新的数组引用**（即使内容相同）
3. `selectedSkills` 引用变化 → `fetchCandidates` 被重新创建
4. `fetchCandidates` 变化 → `useEffect` 触发
5. `useEffect` 执行 → 调用 API → 更新状态 → 触发重新渲染
6. 回到步骤 1，形成**无限循环** ♻️

### 为什么数组引用会变化？

```javascript
// 每次执行都创建新数组
[] === []  // false ❌
[1, 2] === [1, 2]  // false ❌

// React 使用 Object.is() 进行引用比较
// 即使内容相同，不同的数组对象也被认为是"变化"
```

## ✅ 解决方案

### 修复后的代码

```typescript
// ✅ 修复：使用稳定的字符串作为依赖
const selectedSkillsString = searchParams.get("skills") || "";
const selectedSkills = React.useMemo(
  () => selectedSkillsString ? selectedSkillsString.split(",").filter(Boolean) : [],
  [selectedSkillsString]
);

const fetchCandidates = React.useCallback(async () => {
  // ... fetch logic
  if (selectedSkillsString) {
    params.set("skills", selectedSkillsString);
  }
}, [page, pageSize, sortBy, sortOrder, searchKeyword, selectedSkillsString]);
//                                                        ^^^^^^^^^^^^^^^^^^^^
//                                                        稳定的字符串引用！
```

### 关键改进

1. **使用 `selectedSkillsString`（字符串）作为依赖**
   - 字符串是原始类型，值相同则引用相同
   - `"a,b,c" === "a,b,c"` → `true` ✅

2. **使用 `useMemo` 缓存数组**
   - 只有当 `selectedSkillsString` 变化时才重新创建数组
   - 避免每次渲染都创建新数组

3. **在 API 调用中使用字符串**
   - 直接使用 `selectedSkillsString` 而不是 `selectedSkills.join(",")`
   - 减少不必要的数组操作

## 📊 修复效果

### 修复前
```

GET /api/candidates?... 200 in 23ms  ← 无限循环
GET /api/candidates?... 200 in 25ms
GET /api/candidates?... 200 in 35ms
... (持续不断)
```

### 修复后


## 🎓 学习要点

### 1. React Hooks 依赖项规则

**原始类型 vs 引用类型**:
```typescript
// ✅ 好：原始类型作为依赖
const [count, setCount] = useState(0);
useEffect(() => {
  // count 是数字，值相同则引用相同
}, [count]);

// ❌ 坏：对象/数组作为依赖
const filters = { name: "John" };
useEffect(() => {
  // filters 每次都是新对象！
}, [filters]); // 会导致无限循环

// ✅ 好：使用 useMemo 缓存
const filters = useMemo(() => ({ name: "John" }), []);
useEffect(() => {
  // filters 引用稳定
}, [filters]);
```

### 2. useCallback 依赖项最佳实践

```typescript
// ❌ 避免在依赖项中使用数组/对象
const callback = useCallback(() => {
  doSomething(filters);
}, [filters]); // filters 是对象 → 可能导致问题

// ✅ 使用原始值或 useMemo 缓存的值
const filtersString = JSON.stringify(filters);
const callback = useCallback(() => {
  const filters = JSON.parse(filtersString);
  doSomething(filters);
}, [filtersString]); // 字符串稳定
```

### 3. 常见的无限循环场景

**场景 1**: 在 useEffect 中设置状态
```typescript
// ❌ 错误
useEffect(() => {
  setState(calculateValue()); // 每次渲染都设置
}); // 没有依赖数组

// ✅ 正确
useEffect(() => {
  setState(calculateValue());
}, [/* 明确的依赖 */]);
```

**场景 2**: 依赖项包含对象/数组
```typescript
// ❌ 错误
const data = { key: "value" };
useEffect(() => {
  fetchData(data);
}, [data]); // data 每次都是新对象

// ✅ 正确
const dataKey = "value";
const data = useMemo(() => ({ key: dataKey }), [dataKey]);
useEffect(() => {
  fetchData(data);
}, [data]);
```

**场景 3**: useCallback 依赖不稳定的值
```typescript
// ❌ 错误
const onClick = useCallback(() => {
  console.log(items); // items 是数组
}, [items]); // items 每次都是新数组

// ✅ 正确
const itemsString = JSON.stringify(items);
const onClick = useCallback(() => {
  const items = JSON.parse(itemsString);
  console.log(items);
}, [itemsString]);
```

## 🎉 总结

**问题**: `selectedSkills` 数组引用每次渲染都变化  
**原因**: 数组字面量在每次渲染时创建新引用  
**解决**: 使用 `useMemo` 缓存数组，依赖稳定的字符串  
**效果**: 消除无限循环，API 调用正常
