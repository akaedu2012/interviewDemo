# 状态徽章可见性优化

## 问题描述

在候选人卡片视图模式下，右上角的状态徽章（待筛选、初筛通过、面试中等）与背景装饰光晕颜色相近，导致：
- 文字看不清楚
- 徽章背景太透明
- 视觉对比度不足

**影响**: 用户难以快速识别候选人状态

---

## 优化方案

### 1. 增加背景不透明度

**修改前**:
```css
.status-pending {
  bg-slate-500/20  /* 20% 不透明度 */
}
```

**修改后**:
```css
.status-pending {
  bg-slate-700/90  /* 90% 不透明度 */
}
```

### 2. 优化文字颜色

**修改前**:
```css
text-slate-300  /* 较暗的灰色 */
```

**修改后**:
```css
text-slate-100  /* 接近白色 */
```

### 3. 添加视觉效果

在 `CandidateCard.tsx` 中为 Badge 组件添加：
- `backdrop-blur-sm` - 毛玻璃效果
- `shadow-lg` - 阴影效果
- `z-20` - 提高层级（避免被背景遮挡）

### 4. 增强边框对比度

**修改前**:
```css
border-slate-500/30  /* 30% 不透明度 */
```

**修改后**:
```css
border-slate-500/50  /* 50% 不透明度 */
```

### 文件2: `components/candidates/CandidateCard.tsx`

#### Badge 组件样式
```tsx
{/* 修改前 */}
<Badge 
  className={cn("px-3 py-1 text-xs font-medium", statusColors[candidate.status])} 
  variant="secondary"
>
  {candidate.status}
</Badge>

{/* 修改后 */}
<Badge 
  className={cn(
    "px-3 py-1 text-xs font-medium relative z-20 backdrop-blur-sm shadow-lg",
    statusColors[candidate.status]
  )} 
  variant="secondary"
>
  {candidate.status}
</Badge>
```

**新增样式说明**:
- `relative` - 相对定位
- `z-20` - 层级20（高于背景装饰的z-10）
- `backdrop-blur-sm` - 轻微的背景模糊（毛玻璃效果）
- `shadow-lg` - 大阴影（增强立体感）

## 技术实现

### CSS层叠顺序
```
Layer 0: 卡片背景装饰 (无z-index)
Layer 1: 卡片内容 (z-10)
Layer 2: 状态徽章 (z-20) ← 最高层
```

### 毛玻璃效果
```css
backdrop-blur-sm  /* 轻微模糊背后的内容 */
```
作用：增加徽章的独立感，与背景分离

### 阴影效果
```css
shadow-lg  /* 大阴影 */
```
作用：增强立体感和层次感

## 性能影响

### backdrop-filter
- **性能**: 轻微影响（使用了硬件加速）
- **兼容性**: 现代浏览器支持良好
- **优化**: 使用 `backdrop-blur-sm`（小模糊半径）

### shadow-lg
- **性能**: 几乎无影响
- **实现**: 使用 CSS `box-shadow`

**结论**: 性能影响可忽略不计
