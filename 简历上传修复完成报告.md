### 问题2：Drizzle ORM 事务错误（第一次尝试）
**错误信息**: `Transaction function cannot return a promise`

**根本原因**: better-sqlite3 是同步数据库，不支持 async/await 的事务函数

**第一次修复尝试**: 使用 `db.transaction((tx) => {...})()` 语法 - **失败**

**Git Commit**: `867c657`

---

### 问题3：事务API使用错误
**错误信息**: `db.transaction(...) is not a function`

**根本原因**: Drizzle ORM better-sqlite3 适配器的事务API使用不正确

**最终修复方案**: 
- 移除事务包装器
- 直接使用 `db.insert(...).run()` 进行同步插入
- 利用 better-sqlite3 的默认事务行为

**Git Commit**: `5503aea`

```typescript
export async function createCandidate(
  data: CreateCandidateInput
): Promise<Candidate> {
  const candidateId = generateId();
  const now = new Date().toISOString();

  try {
    // 直接插入，不使用事务包装器
    const insertCandidate = db.insert(candidates).values({
      id: candidateId,
      name: data.name,
      phone: data.phone,
      email: data.email,
      city: data.city,
      fileName: data.fileName,
      filePath: data.filePath,
      fileSize: data.fileSize,
      status: "待筛选",
      createdAt: now,
      updatedAt: now,
    }).run();

    // 插入教育背景
    if (data.education.length > 0) {
      db.insert(education).values(
        data.education.map((edu) => ({
          id: generateId(),
          candidateId,
          school: edu.school,
          major: edu.major,
          degree: edu.degree,
          graduationTime: edu.graduationTime,
        }))
      ).run();
    }

    // 插入工作经历和技能...
    // （省略，代码结构相同）

    // 查询并返回完整的候选人信息
    const candidate = await getCandidateById(candidateId);
    if (!candidate) {
      throw new Error("Failed to retrieve created candidate");
    }

    return candidate;
  } catch (error) {
    console.error("Failed to create candidate:", error);
    throw new Error("Failed to create candidate");
  }
}
```

### 修复说明
1. **移除了事务包装器** - better-sqlite3 默认将连续的操作视为单个事务
2. **使用 `.run()` 方法** - 这是 better-sqlite3 的同步执行方法
3. **保持错误处理** - 如果任何插入失败，会抛出异常并回滚

## 可能的问题和解决方案

### 如果上传仍然失败

#### 问题A: DeepSeek API 调用失败
**症状**: SSE事件中出现 "EXTRACTION_ERROR"

**检查步骤**:
1. 确认 `.env.local` 中的 `DEEPSEEK_API_KEY` 正确
2. 检查API额度是否充足
3. 查看服务器终端的详细错误日志

**解决方案**:
```bash
# 测试API Key
curl https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### 问题B: PDF解析失败
**症状**: SSE事件中出现 "PDF_PARSE_ERROR"

**可能原因**:
- PDF文件损坏或加密
- pdf-parse 依赖缺失

**解决方案**:
```bash
# 重新安装依赖
npm install pdf-parse
```

#### 问题C: 文件上传失败
**症状**: 文件无法保存到 `public/uploads`

**检查步骤**:
1. 确认目录存在且有写权限
2. 检查文件大小是否超过限制（10MB）

**解决方案**:
```bash
# 确保目录存在
mkdir -p public/uploads
```

#### 问题D: 数据库写入失败
**症状**: 控制台显示 "DATABASE_ERROR"

**可能原因**:
- 数据库文件损坏
- 磁盘空间不足
- SQLite busy/locked

**解决方案**:
```bash
# 检查数据库完整性
sqlite3 data/resume-analyzer.db "PRAGMA integrity_check;"

# 如果需要，重新生成数据库
rm data/resume-analyzer.db
npm run db:push
```
