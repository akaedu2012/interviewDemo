# 简历上传错误修复说明

## 🐛 问题描述

**症状**:
- 上传简历后，AI 提取进度正常显示（basic → education → experience → skills → complete）
- 但在最后创建候选人记录时失败
- 错误信息：`Extraction error: Failed to create candidate INTERNAL_ERROR`
- EventSource 连接错误

**错误截图分析**:
```
Extraction error: Failed to create candidate INTERNAL_ERROR
EventSource connection error
MessageEvent {isTrusted: true, data: '{"error":"Failed to create candidate","code":"INTERNAL_ERROR"}', ...}
```

## 🔍 问题分析

### 根本原因

`services/candidateManager.ts` 中存在**错误的导入语句**：

```typescript
// ❌ 错误代码
import { generateId, safeJsonParse } from "@/lib/utils";
//                     ^^^^^^^^^^^^^^
//                     这个函数不存在！
```

### 错误流程

1. 用户上传 PDF 简历文件 → ✅ 成功
2. PDF 解析提取文本 → ✅ 成功  
3. AI 提取结构化信息 → ✅ 成功（所有阶段）
4. 调用 `createCandidate()` 保存数据 → ❌ **失败**
   - 模块加载时尝试导入 `safeJsonParse`
   - 函数不存在 → 模块加载失败
   - 抛出 INTERNAL_ERROR
5. SSE 连接中断 → 前端显示错误

### 为什么之前没发现？

- `candidateManager.ts` 在上传简历之前**不会被加载**
- 只有当需要创建候选人时才会导入这个模块
- 导入失败 → 整个服务不可用

## ✅ 解决方案

### 修复代码

```typescript
// ✅ 修复后
import { generateId } from "@/lib/utils";
// 移除不存在的 safeJsonParse 导入
```

### 完整的修复文件

**`services/candidateManager.ts` (第 3 行)**:
```typescript
// 修复前
import { generateId, safeJsonParse } from "@/lib/utils";

// 修复后  
import { generateId } from "@/lib/utils";
```

## 📊 修复效果

### 修复前
```
上传 PDF → 解析成功 → AI 提取成功 → 创建候选人失败 ❌
错误: INTERNAL_ERROR
```

### 修复后
```
上传 PDF → 解析成功 → AI 提取成功 → 创建候选人成功 ✅
候选人记录保存到数据库 → 返回 candidateId → 自动跳转到列表页
```

## 🔍 为什么会有 `safeJsonParse` 导入？

### 可能的原因

1. **代码演进**: 之前可能计划实现这个函数但后来改变了方案
2. **复制粘贴错误**: 从其他文件复制导入语句时带过来的
3. **重构遗留**: 重构代码时忘记清理未使用的导入

### `candidateManager.ts` 中实际不需要 JSON 解析

检查代码发现，`candidateManager.ts` 中**从未使用** `safeJsonParse` 函数：

- 所有数据都是强类型的 TypeScript 对象
- 数据库操作使用 Drizzle ORM，自动处理序列化
- 不需要手动 JSON 解析

## 🎓 学习要点

### 1. 导入语句检查

**问题**:
```typescript
import { func1, func2, func3 } from "./utils";
// 如果 func2 不存在，整个模块加载失败
```

**最佳实践**:
```typescript
// ✅ 只导入实际使用的函数
import { func1, func3 } from "./utils";

// ✅ 使用 ESLint 规则检测未使用的导入
// "no-unused-vars": ["error", { "varsIgnorePattern": "^_" }]
```

### 2. 模块加载顺序影响错误发现

```typescript
// 模块 A
import { nonExistent } from "./utils"; // ❌ 错误

// 但如果 模块 A 从未被导入，错误不会显现
// 只有当某个地方 import 模块 A 时，错误才会暴露
```

**经验**:
- 编写代码后立即测试所有导入路径
- 使用 TypeScript 类型检查 (`npm run type-check`)
- 使用 ESLint 检测未使用的导入

### 3. 错误追踪技巧

**当遇到 INTERNAL_ERROR 时**:

1. **检查服务器日志**: 查找完整的错误堆栈
   ```bash
   # 开发服务器通常会打印详细错误
   ```

2. **逐步追踪调用链**:
   ```
   前端上传 → API 路由 → PDF 解析 → AI 提取 → 数据库保存
                                                      ↑ 
                                                   错误位置
   ```

3. **检查导入语句**: 导入错误是最常见的 INTERNAL_ERROR 原因
