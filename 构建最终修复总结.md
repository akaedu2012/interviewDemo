## 最后两个关键修复

### 1. 修复动态服务器使用错误 ⭐

**错误信息**:
```
Dynamic server usage: Route /api/candidates couldn't be rendered statically 
because it used `request.url`
```

**问题分析**:
- Next.js 14 默认尝试静态渲染所有路由
- API 路由使用 `request.url` 需要在运行时访问请求对象
- 这需要明确标记为动态路由

**修复方案**:
在 `app/api/candidates/route.ts` 文件顶部添加：

```typescript
// 标记此路由为动态路由
export const dynamic = 'force-dynamic';
```

**Route Segment Config 选项**:
- `dynamic = 'force-dynamic'`: 强制动态渲染
- `dynamic = 'force-static'`: 强制静态渲染
- `dynamic = 'error'`: 如果使用动态功能则报错
- `dynamic = 'auto'`: 自动检测（默认）

### 2. 修复 useSearchParams Suspense 边界 ⭐

**错误信息**:
```
⨯ useSearchParams() should be wrapped in a suspense boundary at page "/"
Error occurred prerendering page "/"
```

**问题分析**:
- `useSearchParams()` 是一个动态 API，会导致组件在客户端渲染
- Next.js 需要一个 Suspense 边界来处理加载状态
- 这样可以实现部分预渲染 (Partial Prerendering)

**修复方案**:
将使用 `useSearchParams` 的组件包裹在 Suspense 中：

```typescript
// app/page.tsx

// 1. 将主逻辑提取到 HomePageContent 组件
function HomePageContent() {
  const searchParams = useSearchParams();
  // ... 原有逻辑
}

// 2. 导出的组件包裹 Suspense
export default function Home() {
  return (
    <Suspense fallback={<LoadingSpinner text="加载中..." />}>
      <HomePageContent />
    </Suspense>
  );
}
```

**好处**:
- ✅ 允许页面部分预渲染
- ✅ 提供更好的加载体验
- ✅ SEO 友好

## 关键技术点

### 1. Next.js 14 路由类型

```typescript
// 静态路由 (Static Route)
export default function Page() { ... }

// 动态路由 (Dynamic Route)
export const dynamic = 'force-dynamic';
export default function Page() { ... }

// 使用动态参数
export default function Page({ params }: { params: { id: string } }) { ... }
```

### 2. Suspense 边界最佳实践

```typescript
// ❌ 不推荐：直接使用 useSearchParams
export default function Page() {
  const searchParams = useSearchParams();
  // ...
}

// ✅ 推荐：包裹 Suspense
export default function Page() {
  return (
    <Suspense fallback={<Loading />}>
      <PageContent />
    </Suspense>
  );
}

function PageContent() {
  const searchParams = useSearchParams();
  // ...
}
```

### 3. API 路由配置

```typescript

// 强制动态渲染

  // 可以使用 request.url, request.headers 等
```
