# 构建错误修复说明

## 问题概述
运行 `npm run build` 时遇到多个 TypeScript 类型错误，导致构建失败。

## 修复的错误列表

### 1. Zod 错误对象属性名错误
**错误信息**: `Property 'errors' does not exist on type 'ZodError<...>'`

**问题根因**: Zod 的 `safeParse` 返回的错误对象使用 `issues` 属性，而不是 `errors`

**修复文件**:
- `app/api/candidates/[id]/match/route.ts` - 第51行
- `app/api/candidates/[id]/status/route.ts` - 第78行  
- `app/api/jobs/route.ts` - 第63行

**修复方式**:
```typescript
// 错误写法
const errors = validationResult.error.errors;

// 正确写法
const issues = validationResult.error.issues;
```

### 2. Zod enum errorMap 选项错误
**错误信息**: `errorMap' does not exist in type ...`

**问题根因**: Zod 的 `enum` 类型不支持 `errorMap` 选项

**修复文件**: `app/api/candidates/[id]/status/route.ts` - 第8行

**修复方式**:
```typescript
// 错误写法
z.enum(["待筛选", "初筛通过", "面试中", "已录用", "已淘汰"], {
  errorMap: () => ({ message: "..." })
})

// 正确写法
z.enum(["待筛选", "初筛通过", "面试中", "已录用", "已淘汰"])
```

### 3. Select 组件 onValueChange 类型错误
**错误信息**: `Type 'string' is not assignable to type 'string | null'`

**问题根因**: shadcn/ui 的 Select 组件 `onValueChange` 回调接受 `string | null` 类型

**修复文件**:
- `components/candidates/CandidateSorter.tsx` - 第32行
- `components/candidates/PageSizeSelector.tsx` - 第25行

**修复方式**:
```typescript
// 错误写法
const handleChange = (value: string) => {
  onSortChange(value);
};

// 正确写法
const handleChange = (value: string | null) => {
  if (!value) return;
  onSortChange(value);
};
```

### 4. ARIA 可访问性警告
**错误信息**: `Elements with the ARIA role "combobox" must have the following attributes defined: aria-controls`

**修复文件**: `components/ui/TagInput.tsx` - 第175行

**修复方式**:
```tsx
// 添加 aria-controls 属性
<div
  role="combobox"
  aria-controls="tag-input-list"
  aria-expanded="false"
  aria-haspopup="listbox"
>
```

### 5. Modal DialogTrigger asChild 属性错误
**错误信息**: `Property 'asChild' does not exist on type ...`

**问题根因**: 使用的 Dialog 组件版本不支持 `asChild` 属性

**修复文件**: `components/ui/Modal.tsx` - 第107行

**修复方式**:
```tsx
// 错误写法
<DialogTrigger asChild>{trigger}</DialogTrigger>

// 正确写法
<DialogTrigger>{trigger}</DialogTrigger>
```

### 6. 脚本文件变量重复声明
**错误信息**: `Cannot redeclare block-scoped variable 'BASE_URL'`

**问题根因**: 多个测试脚本文件都声明了 `BASE_URL` 变量，在严格模式下导致重复声明错误

**修复文件**: `tsconfig.json`

**修复方式**:
```json
{
  "exclude": ["node_modules", "scripts"]
}
```
将 `scripts` 目录从 TypeScript 编译中排除

### 7. 数据库插入类型错误
**错误信息**: `Type 'number' is not assignable to type 'boolean'`

**问题根因**: `isActive` 字段应该是 boolean 类型，但传入了数字 `1`

**修复文件**: `scripts/create-mock-candidate.ts` - 第127行

**修复方式**:
```typescript
// 错误写法
isActive: 1,

// 正确写法
isActive: true,
```

### 8. 文件类型验证类型错误
**错误信息**: `Argument of type 'string' is not assignable to parameter of type '"application/pdf"'`

**问题根因**: `ALLOWED_TYPES` 使用 `as const` 定义为只读元组类型，而 `file.type` 是普通 `string`

**修复文件**: `services/resumeUpload.ts` - 第20行

**修复方式**:
```typescript
// 使用类型断言
if (!FILE_UPLOAD.ALLOWED_TYPES.includes(file.type as any)) {
  // ...
}
```

### 3. Select 组件类型辅助函数
```typescript
// lib/form-utils.ts
export function createSelectHandler<T>(
  callback: (value: T) => void,
  parser?: (value: string) => T
) {
  return (value: string | null) => {
    if (!value) return;
    callback(parser ? parser(value) : (value as T));
  };
}
```

### 4. 改进类型安全
对于文件类型验证，可以改进类型定义：
```typescript
// lib/constants.ts
export const FILE_UPLOAD = {
  ALLOWED_TYPES: ["application/pdf"] as readonly string[],
  // 或者使用类型守卫
} as const;

// services/resumeUpload.ts
function isAllowedType(type: string): type is "application/pdf" {
  return FILE_UPLOAD.ALLOWED_TYPES.includes(type);
}

if (!isAllowedType(file.type)) {
  // ...
```
